# Bug B03: Potential IndexError with merge_base

## Severity
**Critical**

## Location
- `src/worktreeflow/wtf.py:714`
- `src/worktreeflow/wtf.py:881`

## Description
The code assumes `merge_base()` always returns at least one result, but with unrelated Git histories, it can return an empty list, causing an IndexError.

## Current Code

### Location 1: sync_main() - Line 714
```python
merge_base = self.repo.merge_base(self.repo.head.commit, upstream_ref.commit)

if merge_base[0] != self.repo.head.commit:  # IndexError if merge_base is empty
    raise GitCommandError(...)
```

### Location 2: zero_ffsync() - Line 881
```python
merge_base = self.repo.merge_base(origin_ref.commit, upstream_ref.commit)

if merge_base[0] != origin_ref.commit:  # IndexError if merge_base is empty
    console.print(f"[red]ERROR: Cannot fast-forward...")
```

## Problem
`GitPython.repo.merge_base()` returns an empty list when commits have completely unrelated histories (no common ancestor). Accessing `merge_base[0]` will raise:

```python
IndexError: list index out of range
```

This can occur when:
- A repository is initialized fresh without proper fork setup
- Branches from different repositories are compared
- History has been rewritten without common base

## Impact
**Application crash** with unhelpful error message instead of clear guidance about unrelated histories.

## Proposed Fix

### Location 1: sync_main()
```python
merge_base = self.repo.merge_base(self.repo.head.commit, upstream_ref.commit)

if not merge_base:
    raise GitCommandError(
        "git merge-base",
        1,
        stderr=f"No common ancestor found between {base} and upstream/{base}\n"
               f"Your repository may have unrelated histories.\n"
               "This usually means the fork was not set up correctly.\n"
               f"  To force-sync (DESTRUCTIVE): wtf sync-main-force --confirm"
    )

if merge_base[0] != self.repo.head.commit:
    raise GitCommandError(...)
```

### Location 2: zero_ffsync()
```python
merge_base = self.repo.merge_base(origin_ref.commit, upstream_ref.commit)

if not merge_base:
    console.print(f"[red]ERROR: No common ancestor between origin/{base} and upstream/{base}[/red]")
    console.print("Your repositories have unrelated histories.")
    console.print("This usually means the fork was not set up correctly.")
    sys.exit(1)

if merge_base[0] != origin_ref.commit:
    console.print(f"[red]ERROR: Cannot fast-forward...")
```
