# Bug B06: Redundant json imports

## Severity
**Medium Priority** - Code Quality Issue

## Location
- `src/worktreeflow/wtf.py:1072`
- `src/worktreeflow/wtf.py:1392`
- `src/worktreeflow/wtf.py:1602`

## Description
The `json` module is imported at the module level (line 23) but is redundantly imported again inside three different functions.


## Problem
While not technically a bug (Python allows re-importing and just returns the cached module), this is:
1. **Inconsistent**: Line 1602 uses `json` without re-importing (correct), while 1072 and 1392 re-import it
2. **Confusing**: Makes it unclear whether `json` is available at module scope
3. **Unnecessary overhead**: Although minimal, re-importing adds unnecessary function calls
4. **Bad practice**: Goes against Python conventions

## Impact
- No functional impact
- Minor code readability and maintainability issue
- Could confuse developers about module availability

Remove the redundant local imports:
